Node-npm使用

一、基于之前学的 node 的基础对比现在的 node 高级来说，只是让我们大概了解下基础。现在我带领大家来安装一下 node 的安装和环境变量的配置的环境，带大家重新整理我们的 node.js 吧
1、首先我们先安装一下我们的 node.js 可以去官网查看最新的 node.js msi 的安装包进行安装，安装过程中，只需要改变一下我们安装目录，一般我们会在 D 盘的盘符中来新建一个 Node 的文件夹，一直 next 安装到里面就可以，
2、现在我们进行验证一下。按住我们键盘上的window+R,会出现一个输入框在里面输入cmd命令符，然后出现的命令符小黑板，在里面输入 node -v 和 npm -v ,看到
我们的版本号之后就代表我们的 node 就已经安装完毕了，由于我们所使用npm是国外的东西，所以我们要使用阿里
淘宝镜像。
1、首先安装 npm i cnpm -g;
2、**npm config set registry https://registry.npm.taobao.org
3、安装淘宝镜像 cnpm config set registry https://registry.npm.taobao.org
4、切换 npm config set registryregistry=https://registry.npmjs.org/
3、环境变量
1、添加环境变量 Path
2、node.js 的安装路径
3、npm 包的下载路径
4、配置环境变量打开环境变量，里面有系统变量和用户变量，这里我们的变量改变我们的系统变量，这里点击我们的新建系统变量会出现二个输入框，第一个输入框是要输入你的NODE_PATH 第二个输入框是要输入我们D:Node\npm\node_modules **进行配置就可以了
5、配置完成后我们需要更改我们的全局包的默认路径 npm config prefix ** 全局安装的存放路径
6、npm config set cache **指定下包的缓存
二、常用命令
一、包管理器
1、npm adduser 登录
2、npm clear cache 清除缓存
3、npm config set 设置配置命令
4、npm config get 读取
**5、npm –helo 查看帮助 **
6、package 代表包的名称
7、-g 全局安装
8、-d 当前路径安装
9、–save 向包管理文件添加 生产环境依赖
10、–save-dev 向包管理文件添加 开发环境依赖
11、npm install -g 全局安装
12、npm uninstall -g 全局卸载
13、npm publish 发布
14、npm unpublish 取消发布
15、npm owner 包的作者
16、npm whoami 用户名
17、npm -v 查看版本号
18、npm run <>执行包管理文件的某条命令
19、npm start 启动默认命令 简写 ,npm run start
20、npm init 生产一个包管理文件
21、npm i 等同于 npm install
