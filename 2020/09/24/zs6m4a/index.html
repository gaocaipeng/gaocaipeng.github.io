网站SEO优化

2020-09-24- 2022-01-05
技术积累-SEO优化

  1. 网站需要使用 ssr 开发模式，spa 单页面开发不利于 SEO
  2. 只解析公司因业务需要决定开通的子域名。未开通子域名一律不解析。不要使用泛解析（以通配符*代替子域名）。
  3. 不能有两个或两个以上域名显示同样网站内容（无论以哪种方式实现，如解析至相同根目录、上传同样文件和数据库）。
  4. 用于测试的子域名使用 robots 文件禁止搜索引擎抓取。
  5. 未完成内容的网站、栏目不得上线。
  6. 不存在页面必须返回 404 代码。每个月使用线上工具确认不存在页面服务器头信息。
  7. 服务器开启 gzip 压缩。
  8. 除了已规划禁索引擎抓取和收录的内容外，网站所有 URL 一律静态化止搜。
  9. 站内搜索结果页面使用 robots 文件禁止搜索引擎抓取。
  10. 网站栏目与 URL 目录需一一对应，一级分类对应一级目录，二级分类对应二级目录。最多分级至二级目录，如果产品/页面数需要三级以上分类，请先与 SEO 部门沟通导航和内部链接解决方案。 h5 上问答模块有三级目录是产品需求
  11. 栏目/目录 URL 以斜线结尾，后面不要加 index.php 之类文件名。
  12. 栏目名、文件名等 URL 中的字母一律小写。除活动页无这种问题
  13. URL 中不要出现除了短横线之外的其它任何特殊字符。
  14. 除栏目需要，URL 中不得添加其它多余目录层次。 除 pc 使用百度统计埋点的页面
  15. 无论中英文网站，URL 目录名使用相应英文单词，长度以 3 个英文单词为限。不使用中文或拼音。也可以考虑使用数字/字母编号。
  16. 英文网站，产品/文章页面文件名使用产品名称/文章标题，去掉虚词。
  17. 中文网站，产品/文章页面文件名使用数字/字母编号。
  18. 同一个产品不要放置于多个分类之下，只置于一个分类。
  19. 所有页面加 canonical 标签。如果不确认 canonical 标签列出的规范化 URL 应该是哪个，请与 SEO 部门咨询。
  20. 所有页面加面包屑导航。
  21. 页面 Title 标签、Deion 标签、H1 文字按格式自动生成缺省版本，但系统需要给 SEO 部门预留人工填写功能。
  22. 一个页面只使用一次 H1。
  23. 栏目页面 Title 缺省格式：二级栏目名称– 一级栏目名称– 网站名称
  24. 栏目页面翻页 Title 缺省格式：二级栏目名称– 一级栏目名称– 网站名称– 第 X 页
  25. 产品页面 Title 缺省格式：产品名称– 网站名称
  26. 栏目页面 H1 缺省格式：一级栏目名称– 二级栏目名称
  27. 产品页面 H1 缺省格式：产品名称
  28. Deion 标签从栏目或产品说明文字第一段截取长度 20 字以上、70 字以下的完整句子。
  29. 产品页面如果有格式化数据，Title 和 Deion 标签可以充分利用，使其组成通顺可读的句子。
  30. 除非 SEO 部门另行要求，页面不使用 Keywords 标签。每个页面 Keywords 都是根据之前 SEO 字段去添加的
  31. 除非 SEO 部门另行要求，所有链接使用 HTML 代码链接，不要使用 JS 生成。
  32. 所有用户生成内容（UGC）中的链接加 nofollow 属性，例如本站点http://www.xminseo.com/就对评论区域的url加了nofollow标签。
  33. 栏目、产品/文章页面主图加 ALT 文字，如没有人工填写，可与页面 Title 相同。
  34. 禁止使用任何隐藏文字或链接。
  35. 尽量避免使用表格（table），尤其是嵌套表格。
  36. 在不明显影响视觉效果前提下，所有图片均需要最大程度压缩处理后才能使用。
  37. 不要使用超出实际显示尺寸的图片（不要上传大尺寸图片，然后缩小尺寸显示）。
  38. 页面纯文字代码（包括 HTML、JS、CSS）不超过 500K。特殊页面如首页可适当放宽。
  39. 删除未使用的 CSS 代码。尽量合并 CSS 文件。
  40. 慎用、少用 JS。在主流浏览器测试 JS 是否拖慢页面打开速度。
  41. 使用主流浏览器实际测试页面打开速度，应不超过 3 秒。条件允许的话，从多省市测试。
  42. 后台实时或定期生成并更新 XML 版 Sitemap，包括首页、栏目及翻页、产品/文章页面。
